# GitHub-Style Alerts

All five alert types supported by GitHub Flavored Markdown.

## Note

> [!NOTE]
> This is a **note** alert. Use it to highlight information that users
> should take into account, even when skimming.
>
> Notes can contain `inline code` and [links](https://example.com).

## Tip

> [!TIP]
> This is a **tip** alert. Use it to provide helpful advice
> for doing things better or more easily.
>
> For example: run `markdown2docx.py --help` to see all options.

## Important

> [!IMPORTANT]
> This is an **important** alert. Use it to convey crucial information
> necessary for users to succeed.
>
> Key details that *must not* be overlooked belong here.

## Warning

> [!WARNING]
> This is a **warning** alert. Use it to indicate urgent information
> that demands immediate user attention to avoid problems.
>
> Proceed with ~~reckless abandon~~ caution.

## Caution

> [!CAUTION]
> This is a **caution** alert. Use it to advise about risks or
> negative outcomes of certain actions.
>
> Potential data loss may occur if you skip this step.
